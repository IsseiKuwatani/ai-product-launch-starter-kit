# 無料キット配布設計

## 推奨方針

現時点では **1つの配布リンク = GitHub Pagesのキットハブ** を推奨します。

理由:

- LINE登録後に案内するURLが1つで済む
- 5つの配布物を同じページから見られる
- GitHub上のマニュアル/Markdown/テンプレートと相性が良い
- 後からLINE Harnessのクリック計測リンクに差し替えやすい

## パーソナライズについて

LINEの表示名で「ようこそ〇〇さん」を出すには、基本的にLIFFログインとプロフィール取得が必要です。

ただし、無料キット配布段階では以下の理由で、まずはクリック計測優先が安全です。

- LIFF導入は初回アクセス時の認可・ログイン導線が増える
- 無料キット閲覧だけなら、ユーザー体験は1リンクの方が軽い
- LINE Harnessの tracked link / リッチメニュークリックでCV計測した方が運用しやすい

## 段階設計

### Phase 1: 今回実装

- LINE自動応答: `無料キット` → GitHub Pagesの配布ページ
- 配布ページ内に5つのキットを一覧表示
- GitHubリポジトリ上にマニュアル・プロンプト・テンプレートを配置

### Phase 2: 計測強化

- LINE Harnessで配布ページURLをtracked link化
- `無料キット`, `キット`, `個別相談` のクリック/メッセージ反応を分けて計測

### Phase 3: パーソナライズ

- LIFF Hubを作る
- LINEプロフィールからdisplayNameを取得
- `ようこそ、〇〇さん` を出す
- 閲覧イベント/完了イベントをD1へ記録
