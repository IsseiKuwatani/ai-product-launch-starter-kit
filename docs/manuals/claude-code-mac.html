1. 概要

Claude Codeは、Claudeをローカルの作業フォルダ上で利用するためのAIコーディングエージェントです。チャットで相談するだけでなく、実際のファイルを読み、コードやドキュメントを作成・修正し、必要に応じてコマンド実行まで行えます。

新規事業開発では、仮説整理、検証LP作成、簡易Webアプリ制作、営業資料作成、FAQ・ナレッジ整備などに使えます。

位置づけ：エンジニア専用というより、事業担当者が検証速度を上げるための実務ツールとして使えます。

2. 動作要件

  • macOS 13.0以降
  • インターネット接続
  • Claudeアカウント
  • Claude Pro / Max等のサブスクリプション、またはAnthropic APIキー
  • macOS標準のターミナル

この手順では、HomebrewやNode.jsなどの別ルートは扱わず、最短の公式インストール手順に絞ります。

3. Macでの導入手順

3.1 ターミナルを開く

command + space → 「ターミナル」 → Enter。

3.2 Claude Codeをインストール

以下の1行をコピーして実行します。

curl -fsSL https://claude.ai/install.sh | bash

インストール後、ターミナルを開き直すか、表示された案内に従ってPATHを反映します。

3.3 動作確認

claude --version

バージョンが表示されればインストール完了です。

4. 初回起動とCLI認証

重要：Claude Codeはインストールだけでは使い始められません。最初にCLI上でログインし、認証状態を確認します。

4.1 ログインを開始

claude auth login

ブラウザが開いたら、Claudeアカウントでログインします。Claude Pro / Max等のサブスクリプションで使う場合は通常ログインで進めます。

4.2 API利用課金で使う場合

Anthropic ConsoleのAPI課金で使う場合は、次のコマンドを使います。

claude auth login --console

会社や組織のSSOを使う場合は、必要に応じて claude auth login --sso を使います。

4.3 認証状態の確認

claude auth status --text

ログイン済みの表示になればOKです。未ログインの場合は、もう一度 claude auth login を実行します。

4.4 診断

claude doctor

最後に作業フォルダで claude を起動し、最初の画面が表示されるか確認します。

claude
注意：APIキーやパスワードは、Slack・チャット・GitHub・ソースコードに貼らないでください。認証はCLIのログイン画面や環境変数で管理します。

5. 基本操作

操作コマンド
起動claude
作業フォルダで相談作業したいフォルダでターミナルを開き、claude
終了/exit または Ctrl + D
状態確認claude doctor
アップデートclaude update または claude upgrade

Claude Codeは、ターミナルを開いているフォルダを作業対象として扱います。別の場所から開いた場合だけ、対象フォルダへ移動してから起動します。

cd ~/Desktop/project-name
claude

6. 権限モード

通常起動

claude

確認を挟みながら進める標準の起動方法です。

Auto mode

claude --permission-mode auto

安全と判断される操作は自動で進め、リスクの高い操作は確認を求めるモードです。

確認を大幅に省略するモード

claude --dangerously-skip-permissions
注意：自分が管理している安全な作業フォルダでのみ使用します。重要な本番環境、顧客データ、秘密情報、削除して困るファイルがある場所では使用しないでください。

7. 使いどころ

  • 新規事業テーマの仮説整理
  • 顧客ヒアリング設計
  • 検証LP・サービスページ作成
  • 営業メール・架電スクリプト作成
  • 小さな社内ツール作成
  • FAQ・ナレッジ整備

8. スターターキットは /setup で開始できます

AI Webサービス制作スターターキットのテンプレートには、初回セットアップ用の手順が同梱されています。テンプレートフォルダをClaude Codeで開いたら、最初に /setup と入力してください。

cd template/next-ai-product-starter
claude
# Claude Code内で /setup
ポイント：/setup は、作りたいサービスについて質問しながら、まず自分のPCで動く1画面MVPへ整える入口です。最初から公開・ログイン・決済・AI API接続まで広げすぎないようにします。

9. 最初のプロンプト例

/setup もし /setup が読み込まれない場合は、次のように依頼してください。 このフォルダでAI WebサービスのMVPを作ります。 .claude/skills/setup/SKILL.md を読んで、まず1画面MVPを作る手順で進めてください。 いきなり大きく作らず、入力フォームと結果表示までを小さく作ってください。

10. 注意点

機密情報：APIキー、パスワード、顧客の個人情報、契約情報、機密ログ、.env ファイルなどは、社内ルールを確認したうえで扱ってください。
  • 重要ファイルを編集する前に、変更方針を提示させる
  • 実プロジェクトではGitやバックアップを使う
  • 顧客向け文面、法務・セキュリティに関わる内容は人間が確認する

11. トラブルシューティング

claude が見つからない

ターミナルを開き直してから確認します。

claude --version

それでも表示されない場合は再インストールします。

curl -fsSL https://claude.ai/install.sh | bash

ログイン状態を確認する

claude auth status --text
claude auth login

診断する

claude doctor
