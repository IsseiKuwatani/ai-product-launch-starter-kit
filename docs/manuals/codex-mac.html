1. 概要

Codex CLIは、OpenAIのAIコーディングエージェントをローカルの作業フォルダで使うためのツールです。Claude Codeと同じく、ファイルを読み、コードやドキュメントを作成・修正し、実装作業を進められます。

このマニュアルはClaude Code導入ガイドと同じ思想で、非エンジニア・事業担当者が迷わず使い始めるための最短手順に絞っています。

2. 動作要件

  • macOS
  • インターネット接続
  • OpenAI / ChatGPTアカウント、またはAPIキー
  • Node.js / npm、またはHomebrew
  • macOS標準のターミナル
推奨：すでにHomebrewを使っている方は brew install --cask codex、そうでない場合は npm install -g @openai/codex が分かりやすいです。

3. Macでの導入手順

方法A: npmでインストール

npm install -g @openai/codex

方法B: Homebrewでインストール

brew install --cask codex

確認

codex --version

バージョンが表示されればインストール完了です。

4. 初回起動と認証

codex

初回起動時にログインや認証の案内が出た場合は、画面に従ってOpenAI / ChatGPTアカウントでログインします。環境によってはAPIキー設定が必要になる場合があります。

注意：APIキーをチャットやコードに直接貼らないでください。必要な場合は、ツールの案内に従ってローカル環境変数や認証ファイルで管理します。

5. 基本操作

操作コマンド
起動codex
1回だけ依頼して実行codex exec "トップページを改善してください"
作業フォルダで使うcd ~/Desktop/project-name → codex
バージョン確認codex --version

CodexはGitリポジトリ内で使うと変更差分を管理しやすくなります。最初にGitHubテンプレートを複製して使うのがおすすめです。

6. スターターキットは /setup 相当の手順で開始できます

CodexではClaude Codeのように /setup をslashコマンドとして使わない場合でも、同じセットアップ方針で進められます。テンプレートフォルダでCodexを開き、ヘルプページ・プロンプト集・setup skillのファイルパスを渡してください。

cd template/next-ai-product-starter
codex
最初に渡すファイル：docs/help.html / prompts/02-development-prompts.md / 必要に応じて skills/setup/SKILL.md

7. 最初のプロンプト例

このフォルダでAI WebサービスのMVPを作ります。 docs/help.html と prompts/02-development-prompts.md を読んで、まず1画面MVPを作る手順で進めてください。 必要なら skills/setup/SKILL.md も参考にしてください。 いきなり大きく作らず、トップページ、入力フォーム、結果表示の順に小さく進めてください。
ポイント：Codexはファイルを読めるので、ヘルプページやプロンプト集のリンク/ファイルパスを渡すと、その手順に沿って進めやすくなります。

8. 使いどころ

  • Next.jsテンプレートの初期改修
  • LPコピーと画面改善
  • フォームや結果ページの実装
  • READMEや仕様書の整備
  • エラー調査と修正

9. トラブルシューティング

codex が見つからない

ターミナルを開き直してから確認します。

codex --version

npmで失敗する

Node.js/npmの導入状態を確認します。

node --version
npm --version

認証で止まる

画面のログイン案内を確認し、必要に応じてOpenAI / ChatGPTアカウントまたはAPIキーの設定を行います。
