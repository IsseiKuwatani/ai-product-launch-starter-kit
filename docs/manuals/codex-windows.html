1. 概要

Codex CLIは、OpenAIのAIコーディングエージェントをローカルの作業フォルダで使うためのツールです。Claude Codeと同じく、ファイルを読み、コードやドキュメントを作成・修正し、実装作業を進められます。

このマニュアルはClaude Code導入ガイドと同じ思想で、非エンジニア・事業担当者が迷わず使い始めるための最短手順に絞っています。

Windowsでは、PowerShell上で直接使う方法と、WSL2 Ubuntu上で使う方法があります。AI Webサービス制作では、開発コマンドの安定性を考えるとWSL2もおすすめです。

2. 動作要件

  • Windows 10/11
  • インターネット接続
  • OpenAI / ChatGPTアカウント、またはAPIキー
  • Node.js / npm
  • PowerShell、またはWSL2 Ubuntu

3. PowerShellでの導入手順

3.1 Node.jsを確認

node --version
npm --version

表示されない場合は、Node.js公式サイトからLTS版をインストールしてからPowerShellを開き直します。

3.2 Codexをインストール

npm install -g @openai/codex

3.3 確認

codex --version

4. WSL2で使う場合

WSL2 Ubuntu上で使う場合は、Ubuntu内でNode.js/npmを準備してから以下を実行します。

npm install -g @openai/codex
codex --version

Claude Codeと同様、慣れるまでは ~/projects 配下に作業フォルダを作ると安定しやすいです。

5. 初回起動と認証

codex

初回起動時にログインや認証の案内が出た場合は、画面に従ってOpenAI / ChatGPTアカウントでログインします。

注意：APIキーをチャット、GitHub、ソースコードに直接貼らないでください。

6. 基本操作

操作コマンド
起動codex
1回だけ依頼して実行codex exec "READMEを整理してください"
作業フォルダで使うcd project-name → codex
確認codex --version

7. 最初のプロンプト例

このフォルダでAI WebサービスのMVPを作ります。 docs/help.html と prompts/02-development-prompts.md を読んで、まず1画面MVPを作る手順で進めてください。 必要なら skills/setup/SKILL.md も参考にしてください。 作業は小さく区切り、変更後の確認方法も教えてください。
ポイント：Codexはファイルを読めるので、ヘルプページやプロンプト集のリンク/ファイルパスを渡すと、その手順に沿って進めやすくなります。

8. トラブルシューティング

codex が見つからない

PowerShellまたはUbuntuを開き直して確認します。

codex --version

npm installで失敗する

node --version
npm --version

Node.jsを入れ直したあと、PowerShellを開き直します。

パスや権限で詰まる

Windows側の深いフォルダではなく、WSL2の ~/projects 配下で試すと切り分けしやすいです。
