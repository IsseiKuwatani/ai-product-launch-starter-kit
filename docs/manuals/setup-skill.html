/setup はClaude Code向けの初回案内係です。 テンプレート内のファイルを確認し、質問に答えながら、まずは自分のPCで動く簡単なWebサービスまで進められます。Codexを使う人は、このページやskillファイルのリンク/パスをCodexに渡して「この手順で進めて」と頼めます。迷う場合は「開発プロンプト集 順番に貼る版」をStep 0から使ってください。

setup skillを見る テンプレートを見る ヘルプFAQを見る

※ GitHub Pages上では中身を閲覧できます。実際に使うときはテンプレートを自分のPC/リポジトリで開いてください。

使い方

テンプレートを開く

Claude Codeで template/next-ai-product-starter を開きます。

/setup を実行

チャット入力欄で /setup と入力します。Claude Codeがセットアップ用の手順を読み込みます。

いくつかの質問に答える

サービス名、対象ユーザー、課題、入力内容、返したい結果に答えれば、最小MVPへ編集できます。

起動確認

npm run dev でローカル起動し、フォーム入力からモック結果表示まで確認します。

プロンプト集との違い

/setup

Claude Codeではslashとして実行できます。Codexではファイル/ページを読ませる初回手順として活用できます。

プロンプト集

Claude Code / Codexどちらでも使える、順番に貼って進める台本です。

Claude Codeの人はまず /setup。Codexの人はこの説明ページや skills/setup/SKILL.md を読ませるか、プロンプト集 Step 0 から進めてください。

コピペ用コマンド

cd template/next-ai-product-starter
npm install
claude
# Claude Code内で /setup
テンプレートを開いたら、まずは /setup を実行してください。追加の資料を探し回らなくても、質問に答えるだけで、最初の動く形まで進められます。

/setupがやること

状態確認

Node/npm、主要ファイル、依存関係を確認します。

最小ヒアリング

作りたいサービスをMVPに落とし込むため、必要な質問だけします。

画面編集

トップコピー、フォーム、モック結果をテーマに合わせます。

起動確認

ローカルで見える状態まで確認し、次の改善案を3つ出します。

確認ポイント

  • テンプレート内に .claude/skills/setup/SKILL.md がある
  • 念のため .claude/commands/setup.md も入れている
  • GitHub Pagesからもsetup skillの中身を閲覧できる
  • ダウンロード前でも「何をするセットアップか」が分かる
