1. 概要

Windowsでは、Claude Codeを安定して使うために、まずWSL2（Windows上のLinux環境）で使う構成を推奨します。

Claude Codeは作業フォルダのファイルを読み書きしながら進めるツールです。Windows本体のフォルダでも扱えますが、開発系のコマンドはWSL2上の方が安定しやすいです。

2. 動作要件

  • Windows 10/11
  • WSL2が利用できる環境
  • Ubuntu
  • インターネット接続
  • Claudeアカウント
推奨：Windowsネイティブではなく、WSL2 Ubuntu上でClaude Codeを導入します。

3. WSL2 / Ubuntuを準備

3.1 PowerShellを管理者として開く

Windowsメニューで「PowerShell」を検索し、「管理者として実行」を選びます。

3.2 WSLをインストール

wsl --install

再起動を求められた場合は再起動します。

3.3 Ubuntuを起動

スタートメニューから「Ubuntu」を開き、初回ユーザー名とパスワードを設定します。

3.4 Ubuntuを更新

sudo apt update
sudo apt upgrade -y

4. Claude Codeをインストール

Ubuntu上のターミナルで以下を実行します。

curl -fsSL https://claude.ai/install.sh | bash

インストール後、Ubuntuターミナルを開き直して確認します。

claude --version

5. 初回起動と認証

claude

ブラウザ認証が表示された場合は、画面の案内に従ってログインします。

claude auth status --text
claude doctor

6. 作業フォルダの考え方

最初はWSL2内のホームディレクトリに作業フォルダを作るのが安全です。

mkdir -p ~/projects/my-ai-product
cd ~/projects/my-ai-product
claude

Windows側のファイルを扱いたい場合は、/mnt/c/Users/... のようなパスからアクセスできます。ただし、慣れるまではWSL2内に作業用フォルダを作る方がトラブルが少ないです。

7. 基本操作

操作コマンド
起動claude
終了/exit または Ctrl + D
自動寄りで進めるclaude --permission-mode auto
診断claude doctor

9. トラブルシューティング

WSLが起動しない

wsl --status
wsl --update

claude が見つからない

Ubuntuターミナルを開き直してから確認します。

claude --version

権限やパスで詰まる

Windows側の深いフォルダではなく、まず ~/projects 配下に作業フォルダを作って試してください。
