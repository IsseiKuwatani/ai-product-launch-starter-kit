# Setup Skill — Next.js AI Product Starter

## 使う場面

ユーザーがこのテンプレートをClaude Codeで開き、`/setup` または「セットアップして」と依頼したときに使う。
目的は、ダウンロード済みのテンプレートをその場で起動し、ユーザーの事業アイデアに合わせて最小MVPへ整えること。

## 最初に伝えること

このテンプレートは完成品ではなく、AI Webサービスの最初の形を作るための土台です。
いきなり大規模に作らず、まず「誰に何を入力してもらい、どんな結果を返すか」が見える1画面MVPを作ります。

## セットアップ手順

### 1. 現在状態の確認

まず以下を確認する。

```bash
pwd
ls
node -v
npm -v
```

次に主要ファイルを読む。

- `package.json`
- `app/page.tsx`
- `components/ProductForm.tsx`
- `app/globals.css`
- `README.md`

### 2. 依存関係のインストール

`node_modules` が無ければ実行する。

```bash
npm install
```

### 3. ユーザーに3つだけ質問する

一度に細かく聞きすぎない。最初はこれだけ確認する。

1. 作りたいサービス名は何ですか？
2. 誰のどんな課題を解決しますか？
3. ユーザーに何を入力してもらい、どんな結果を返したいですか？

回答が曖昧でも止まらず、仮説として整理して進める。

### 4. 1画面MVPへ編集する

以下をユーザーのテーマに合わせて変更する。

- `app/page.tsx`: ヒーローコピー、説明文、CTA、利用ステップ
- `components/ProductForm.tsx`: 入力項目、ボタン文言、モック結果
- `app/globals.css`: 必要に応じて色・余白・スマホ表示

重要ルール:

- 1回目はAI API接続まで広げすぎない
- まずモック結果で「体験」が伝わる状態にする
- APIキーや秘密情報は絶対にコードに直書きしない
- `.env.local` が必要な場合だけ作る

### 5. 起動して確認する

```bash
npm run dev
```

ブラウザで以下を確認する。

- `http://localhost:3000` が開く
- スマホ幅でも崩れない
- フォーム入力 → 結果表示まで動く
- CTA文言が具体的
- ユーザーが次に何をすればいいか分かる

### 6. 次の改善案を3つだけ出す

最後に、次に進めるなら何をするかを3つに絞って提案する。

例:

1. AI APIを接続して本物の結果生成にする
2. 相談・予約・LINE登録などの導線を追加する
3. 利用者の入力内容を保存するDBを追加する

## 完了条件

- ローカルで起動できる
- ユーザーのサービス名・対象者・課題が画面上に反映されている
- 入力フォームとモック結果が動く
- READMEに起動方法と次にやることが書かれている

## 注意

- `npm run build` が通るか確認できる場合は確認する
- エラーが出たら、まずエラーメッセージを読み、最小修正で直す
- 大量の機能追加や外部サービス連携は、MVP確認後に分けて提案する
