# Setup Skill — Next.js AI Product Starter

## 使う場面

ユーザーがこのテンプレートをClaude Codeで開き、`/setup` または「セットアップして」と依頼したときに使う。
目的は、ダウンロード済みのテンプレートをその場で起動し、ユーザーの事業アイデアに合わせて**ローカルで動く簡単なWebサービスMVP**へ整えること。

## 重要方針

- 外部サービス、本格的なデータベース、ログイン、決済は、ユーザーが明示的に依頼するまで導入しない。
- まずは `npm run dev` でローカルPC上で動く状態を作る。
- 最初は「説明 → 入力 → モック結果 → 次の行動」が見えることを優先する。
- 保存が必要な場合は、まず `localStorage` / JSONダウンロード / CSVダウンロードで対応する。
- AI API接続は後工程。最初はモック結果で体験を確認する。
- APIキーや秘密情報は絶対にコードへ直書きしない。

## 最初に伝えること

このテンプレートは完成品ではなく、AI Webサービスの最初の形を作るための土台です。
いきなり大規模に作らず、まず「誰に何を入力してもらい、どんな結果を返すか」が見えるMVPを作ります。
外部サービス連携は、画面と体験が固まってから必要なものだけ追加します。

## セットアップ手順

### 1. 現在状態の確認

まず以下を確認する。

```bash
pwd
ls
node -v
npm -v
```

次に主要ファイルを読む。

- `package.json`
- `app/page.tsx`
- `components/ProductForm.tsx`
- `app/globals.css`
- `README.md`

### 2. 依存関係のインストール

`node_modules` が無ければ実行する。

```bash
npm install
```

### 3. ユーザーに5つだけ質問する

一度に細かく聞きすぎない。最初はこれだけ確認する。

1. 作りたいサービス名は何ですか？
2. 誰のどんな課題を解決しますか？
3. ユーザーに何を入力してもらいますか？
4. どんな結果を返したいですか？
5. 最後にユーザーに何をしてほしいですか？（保存、コピー、相談、もう一度試す等）

回答が曖昧でも止まらず、仮説として整理して進める。

### 4. MVP方針を短く出す

実装前に以下を短く提示する。

- 今回作る1画面/3画面の範囲
- 作らないもの（ログイン、外部DB、決済、デプロイ等）
- 入力項目
- 結果表示
- 確認方法

### 5. 1画面MVPへ編集する

以下をユーザーのテーマに合わせて変更する。

- `app/page.tsx`: ヒーローコピー、説明文、CTA、利用ステップ
- `components/ProductForm.tsx`: 入力項目、ボタン文言、モック結果、エラー文言
- `app/globals.css`: 色、余白、スマホ表示、カードUI
- `README.md`: 起動方法、使い方、編集ポイント

重要ルール:

- 1回目はAI API接続まで広げすぎない
- まずモック結果で「体験」が伝わる状態にする
- 保存が必要ならまずlocalStorage/JSON/CSVで対応する
- APIキーや秘密情報は絶対にコードに直書きしない
- `.env.local` はAI API追加時など必要な場合だけ作る

### 6. 起動して確認する

```bash
npm run dev
```

ブラウザで以下を確認する。

- `http://localhost:3000` が開く
- スマホ幅でも崩れない
- フォーム入力 → 結果表示まで動く
- 空欄時の案内がある
- CTA文言が具体的
- ユーザーが次に何をすればいいか分かる

### 7. 次の改善案を3つだけ出す

最後に、次に進めるなら何をするかを3つに絞って提案する。
外部サービス導入ありきにしない。

例:

1. モック結果の質を上げて、より実用的な診断/提案にする
2. 結果をコピー・JSON・CSVで保存できるようにする
3. 必要になった段階でAI APIを接続する

## 作れる簡単なサービス例

- 入力内容から改善提案を出す診断サービス
- 商品説明文/求人文/営業文のたたき台を作るサービス
- チェックリストを自動生成するサービス
- 問診内容から次にやることを整理するサービス
- URLや文章を貼って改善点を出すレビューサービス
- 条件を入力して比較表を作るサービス

## 完了条件

- ローカルで起動できる
- ユーザーのサービス名・対象者・課題が画面上に反映されている
- 入力フォームとモック結果が動く
- 空欄/エラー時の表示がある
- スマホで大きく崩れない
- READMEに起動方法と次にやることが書かれている
- 本格的なデータベース、ログイン、決済などが勝手に追加されていない

## 注意

- `npm run build` が通るか確認できる場合は確認する
- エラーが出たら、まずエラーメッセージを読み、最小修正で直す
- 大量の機能追加や外部サービス連携は、MVP確認後に分けて提案する
