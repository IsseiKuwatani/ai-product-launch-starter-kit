1. 使い方

このページは、Claude Code / Codex に貼って使える開発プロンプト集です。

最初から全部を貼るのではなく、作業フェーズに合うものを1つずつ貼ってください。非エンジニアの方は「いきなり編集せず、まず方針を出して」と添えると安全です。

2. MVP設計プロンプト

あなたはAIプロダクトのPM兼エンジニアです。 以下のサービス案を、8週間で最初の公開版まで作る前提でMVPに絞ってください。 # サービス案 [ここに記入] # 想定ユーザー [ここに記入] # ユーザーの悩み [ここに記入] # 使えるAI/API/データ [ここに記入] 出力してください: 1. MVPの目的 2. 最初に作るべき3画面 3. 最初に作るべき5機能 4. 作らないことリスト 5. 2週間ごとの実装計画

3. 画面設計プロンプト

以下のMVPをもとに、Webアプリの画面設計をしてください。 # MVP [ここにMVP概要] # 必須画面 - トップページ - 入力フォーム - 結果ページ - 問い合わせ/相談導線 出力してください: - 各画面の目的 - 表示する情報 - ボタン文言 - 入力項目 - ユーザー導線 - Next.jsでのファイル構成案

4. 実装開始プロンプト

このリポジトリで、Next.jsのAI WebサービスMVPを作ります。 まず以下を実装してください。 # 作りたいもの [サービス名と概要] # 最初の実装範囲 - トップページ - 入力フォーム - 結果ページの仮表示 # デザイン - 白背景 - 濃紺とグリーン - 余白多め - 日本語の太ゴシック - スマホで見やすい # 注意 - まずはAPI接続なしでモックで動かす - 各ファイルの役割を説明する - 実装後に確認手順を教える

5. AI機能追加プロンプト

現在のモック結果ページに、AI生成処理を追加したいです。 # 入力項目 [入力項目を書く] # AIに生成してほしい結果 [診断/提案/文章生成など] # 期待する出力形式 [箇条書き、JSON、表など] まず、API接続を安全に追加する設計を提案してください。 その後、最小実装を行ってください。 .env.local を使い、APIキーをコードに直接書かないでください。

6. LP改善プロンプト

このAI WebサービスのLPを改善してください。 # ターゲット [誰向けか] # ベネフィット [何が嬉しいか] # 不安 [ユーザーが不安に思うこと] # CTA [無料診断/相談予約/LINE登録など] 改善してください: 1. ファーストビュー見出し 2. サブコピー 3. CTAボタン 4. 課題セクション 5. 事例/使い方セクション 6. FAQ 10個

7. デバッグ依頼プロンプト

以下のエラーを解決してください。 # 起きていること [現象] # 実行したコマンド [コマンド] # エラーメッセージ [エラー全文] # 期待する動き [期待] お願い: - 原因候補を3つ出す - 最も可能性が高い原因から確認する - 修正前にどのファイルを見るべきか教える - 修正後の確認コマンドも出す

8. レビュープロンプト

現在の変更内容をレビューしてください。 観点: - ユーザーに価値が伝わるか - MVPとして作りすぎていないか - スマホで使いやすいか - エラー時の表示があるか - CTAが明確か - APIキーや秘密情報がコードに入っていないか 出力: 1. 良い点 2. 問題点 3. 優先修正TOP5 4. そのまま貼れる修正プロンプト
