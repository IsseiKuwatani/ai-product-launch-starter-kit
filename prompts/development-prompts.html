使い方

  1. テンプレートをClaude Code / Codexで開く
  2. Step 0から順番に貼る
  3. 各Stepの最後に出る「次に貼る引き継ぎメモ」を、次のStepへ貼る
  4. Step 4まで終われば、最初のMVPは一度完成
  5. Step 5以降は必要な人だけ追加する

このプロンプト集のゴール

  • 誰向けのサービスか分かる
  • 入力フォームがある
  • 入力内容に応じた結果が表示される
  • スマホでも大きく崩れない
  • 次に何をすればいいか分かる
  • 余計な機能を増やしすぎない

Step 0. 進め方を固定する

最初に一度だけ貼ってください。AIが勝手に大きく作りすぎるのを防ぎます。

あなたは、非エンジニアの事業担当者と一緒に、小さなWebサービスを作るPM兼エンジニアです。

このプロジェクトでは、まず自分のPCで動く1画面MVPを作ります。

ルール:
- まずは「入力する → 結果が出る」体験を完成させる
- 最初はAI接続せず、モック結果で動かす
- ログイン、決済、本格的な保存機能、公開設定は勝手に追加しない
- 1回の変更は小さくする
- 実装前に方針を短く説明する
- 実装後に確認手順を出す
- APIキーや秘密情報はコードに書かない
- 難しい言葉を使いすぎず、初心者にも分かる説明にする

完成条件:
- 入力フォームがある
- 結果が表示される
- スマホでも見やすい
- エラー時に何をすればいいか分かる
- 次に改善する候補が分かる

以後、このルールを守って進めてください。

次は Step 1 に進んでください。

Step 1. アイデアをサービス化する

作りたいものがまだ曖昧な状態で使います。アイデアを「入力 → 結果」の形に整理します。

以下のアイデアを、簡単なWebサービスにできる形へ整理してください。

# アイデア
[ここに作りたいサービスのアイデアを書く]

# 誰向けか
[例: 個人事業主 / 採用担当者 / 店舗オーナー / 学習者]

# 解決したい悩み
[例: 何から始めればいいか分からない / 作業に時間がかかる]

出力してください:
1. サービス名案 5個
2. 一言説明
3. 想定ユーザー
4. ユーザーのBefore / After
5. ユーザーが入力する内容
6. 画面に表示する結果
7. 最初に作る1画面の構成
8. 今回は作らないこと
9. 完成したと言える条件
10. 次のプロンプトに貼るための「引き継ぎメモ」

注意:
- 最初から大きく作りすぎないでください
- 1画面で体験できる形にしてください
- 難しい専門用語は避けてください

次は、出力された 引き継ぎメモ をコピーして Step 2 に貼ってください。

Step 4. 見た目と使いやすさを整える

Step 3で作った画面を、人に見せやすい状態へ整えます。LP改善・フォーム改善・結果画面改善をここにまとめています。

現在のWebサービスを、ユーザーが使いやすく、価値が伝わる形に改善してください。

改善観点:
- 最初の画面で「誰向けのサービスか」が分かるか
- 何を入力すればいいか迷わないか
- 入力項目が多すぎないか
- ボタン文言が分かりやすいか
- 結果が役に立ちそうに見えるか
- 次に何をすればいいか分かるか
- スマホで見やすいか

実装してください:
1. 見出しと説明文の改善
2. 入力フォームの改善
3. ボタン文言の改善
4. 結果表示の改善
5. エラー文言の追加
6. スマホ表示の調整

ルール:
- 新しい大きな機能は足さない
- まず今ある画面を磨く
- 変更後に確認手順を出す

最後に、完成前チェックに使える「引き継ぎメモ」を出してください。

ここまで終われば、最初のMVPは一度完成です。次は Step 7 で完成前チェックをしてください。保存やAI機能が必要な場合だけ、Step 5 / Step 6 に進みます。

Step 7. 完成前チェックをする

人に見せられる状態か確認します。最後に必ず使ってください。

このMVPが、人に見せられる状態になっているか確認してください。

チェック項目:
1. 起動できるか
2. 何ができるサービスかすぐ分かるか
3. 入力できるか
4. 結果が表示されるか
5. エラー時に迷わないか
6. スマホで見やすいか
7. 余計な機能が入っていないか
8. 秘密情報がコードに入っていないか
9. READMEを見て起動できるか
10. 次にやることが分かるか

出力してください:
- 合格 / 要修正
- 良い点
- 直すべき点
- 優先修正TOP5
- そのまま貼れる修正プロンプト
- READMEに書くべき内容

「要修正」が出た場合は、修正プロンプトをそのまま貼って直してください。

困った時. エラー相談テンプレート

途中でエラーが出た時だけ使います。

以下のエラーを直してください。
いきなり大きく書き換えず、原因を特定してから最小修正してください。

# 起きていること
[例: npm run devでエラーが出る / ボタンを押しても結果が出ない]

# エラーメッセージ
[ここにエラーメッセージをそのまま貼る]

# 直前にやったこと
[ここに書く]

お願い:
1. 原因の候補を3つ出す
2. 一番可能性が高い原因を選ぶ
3. 修正するファイルを示す
4. 最小限の修正をする
5. 確認手順を出す
